Python 和 Go
為什麼要分家？

Kuse 的後端有兩個 Repo，一個用 Python 寫，一個用 Go 寫。
它們各自負責完全不同的事情！讓我們來看看為什麼。

🐍

kuse_ai (Python)

「全能大腦」— AI 智慧、畫布、檔案處理、圖片生成、付款

FastAPI + MySQL + Redis 🐹

kuse-ai-go (Go)

「超級快遞員」— 檔案系統、聊天路由、Bot API、即時通訊

Gin + MySQL + Redis + K8s

📖 第一章

為什麼要分成兩個 Repo？

想像你經營一間超棒的餐廳。你需要一位「主廚」負責研發料理，還需要一位「外場經理」負責快速上菜和服務。這兩個角色需要的技能完全不同！

🧑‍🍳 🧪 🎨

kuse_ai = 主廚（Python）

主廚負責所有需要「思考」和「創造」的事情： 研發新菜色（AI 對話、圖片生成）、 處理食材（文件解析、PDF 提取）、 管理菜單和收銀（帳戶系統、Stripe 付款）。 Python 很擅長這些「動腦筋」的工作！

🏃‍♂️ 📁 💬

kuse-ai-go = 外場經理（Go）

外場經理負責所有需要「快速」和「大量處理」的事情： 快速送餐（檔案讀寫、FUSE 檔案系統）、 同時服務很多客人（Bot API、WebSocket）、 管理機器人員工（OpenClaw Pod）。 Go 語言超級快，最適合這種工作！

為什麼不用一種語言就好？

🐍 Python 的強項是 AI/ML 生態系 — LangChain、OpenAI SDK、Anthropic SDK、圖片處理（Pillow）、文件解析（PyPDF）全都有現成的 Python 套件。要做 AI 功能，Python 是最方便的選擇。

🐹 Go 的強項是 高併發 + 系統級操作 — 一個 Go 程式可以同時處理數萬個連線，而且直接操作 Kubernetes API、建立 FUSE 檔案系統這類底層工作，Go 比 Python 快 10-50 倍。

💡 結論：讓 Python 專注做 AI 的事，讓 Go 專注做快速和底層的事，這就是「分工合作」的智慧！

🐍 第二章

kuse_ai：全能大腦（Python）

這是 Kuse 的「大腦」— 所有需要智慧和創造力的工作都在這裡完成。它用 FastAPI 框架打造，連接了超過 10 種外部 AI 服務。

💬Magic Block 系統 — AI 魔法方塊 AI▼
這是 Kuse 最核心的功能！使用者在畫布上選一個東西，點「Magic」，AI 就會幫你做事：
  • 文字生成 — 寫文案、摘要、翻譯
  • 圖片生成 — 從文字描述變出圖片（DALL-E、Flux、Stable Diffusion）
  • 圖片編輯 — 去背、移除物件、替換元素
  • 資料分析 — 從數據生成圖表
  • 網路搜尋 — 即時搜尋網路資訊
# Python 處理 Magic Block 請求 async def execute_magic_block(request): # 1. 判斷要用哪種 AI if request.type == "text_gen": result = await gpt4.chat(prompt, stream=True) elif request.type == "image_gen": result = await flux.generate(prompt, style) # 2. 扣除 credits await deduct_credits(user, cost) # 3. 串流回傳結果 return StreamingResponse(result)
🧠多 LLM 整合 — 語言模型軍團 Python▼
Python 後端連接了所有主流的 AI 模型：
  • GPT-4 / GPT-4o（via Azure OpenAI）— 主力文字模型
  • Claude（Anthropic）— 進階推理
  • Gemini（Google）— 多模態
  • Deepseek — 開源替代方案
  • Flux / DALL-E / Stable Diffusion — 圖片生成
  • Azure Embedding — 語義搜尋向量化
為什麼 Python？因為 LangChain、OpenAI SDK、Anthropic SDK 這些 AI 套件在 Python 上最成熟！
🔍向量搜尋 — Pinecone 語義檢索 AI▼
  • 把文件和圖片描述轉成「向量」（一串數字）
  • 存進 Pinecone 向量資料庫
  • 搜尋時用「語義相似度」而非關鍵字比對
  • 可以問「跟這張圖片類似的有哪些？」
📑文件提取 Pipeline — 從 PDF 到純文字 Python▼
Python 的文件解析能力是 Go 做不到的：
  • doc_extraction_task — 通用文件提取（Unstructured API + LangChain）
  • pdf_extraction_task — PDF 文字/表格提取
  • pdf_to_image_task — PDF 轉圖片預覽
  • img_describing_task — 用 Vision API 描述圖片內容
  • 支援：PDF、Word、Excel、Markdown、CSV、HTML
這些都是用 ARQ 背景任務佇列非同步處理的！
🖼️圖片處理 — 去背、修圖、搜圖 ▼
  • 去背（rmbg_task）— 移除圖片背景
  • 移除元素（rm_elem_task）— 用 AI 修掉不要的東西
  • 替換元素（ch_elem_task）— 把圖片裡的東西換掉
  • 圖片搜尋（image_search_task）— 搜尋網路上相似的圖片
💰Credits 系統 + Stripe 計費 ▼
  • Credit 餘額 — 免費用戶 100 credits，每天 +10
  • Pro 訂閱 — 透過 Stripe 月/年付
  • Student 方案 — 學生特惠價格
  • Team 方案 — 團隊批量訂閱
  • 禮品碼 — 兌換額外 credits
  • Webhook 處理 — 自動處理 Stripe 付款事件
每次使用 AI 功能都會扣除對應的 credits，不同模型價格不同。
🎨Board 白板系統 ▼
  • Board CRUD — 建立、編輯、刪除白板
  • 權限管理 — 私人 / 公開閱讀 / 公開編輯
  • 即時協作 — WebSocket 同步編輯
  • 快照版本 — 自動儲存歷史版本
  • 分享連結 — 靜態分享 URL
  • Board 資料以 JSON 格式存在 MySQL 和 Redis 裡
🔐帳戶與認證系統 ▼
  • Google OAuth — 一鍵登入
  • 帳密註冊 — Email + 密碼
  • JWT Token — 自訂 JWT 認證
  • 多租戶 — 每個使用者可以有多個工作空間
  • 角色管理 — admin / editor / viewer
  • JWKS 端點 — /.well-known/jwks.json 供其他服務驗證
Python 後端是「身份的唯一來源」，Go 後端信任 Python 發出的 Token。
🐹 第三章

kuse-ai-go：超級快遞員（Go）

這是 Kuse 的「行動派」— 所有需要高速處理、大量連線、底層系統操作的工作都在這裡。它是一個包含多個微服務的 Monorepo。

📁Folder Service — 檔案帝國的管理者 Go 主服務▼
這是 Go Repo 裡最大的服務，負責所有檔案相關的操作：
  • 檔案 CRUD — 上傳、下載、移動、複製、刪除
  • 版本管理 — 每個檔案自動保留歷史版本
  • 資源回收桶 — 刪除的檔案可以恢復
  • 檔案分享 — 權限管理（私人/公開）
  • Metadata 管理 — 批量更新檔案屬性
  • 使用量統計 — 追蹤儲存空間用量
// Go 處理檔案上傳 — 比 Python 快很多！ func UploadFile(c *gin.Context) { file, _ := c.FormFile("file") // 1. 計算 SHA256（避免重複上傳） hash := sha256Sum(file) // 2. 上傳到 Google Cloud Storage gcsPath := uploadToGCS(file, hash) // 3. 儲存 metadata 到 MySQL db.Create(&FileModel{Path: path, GCSPath: gcsPath}) }
💬Conversation Service — 對話大管家 Go▼
管理所有 AI 對話：
  • 對話 CRUD — 建立、更新、刪除對話
  • 訊息儲存 — 完整保留聊天記錄
  • 三層壓縮架構 — Go 的獨特設計！
    • 🟢 Recent — 最近的訊息，原文保留
    • 🟡 Summary — 較舊的訊息，壓縮成摘要
    • 🔴 Archive — 最老的訊息，高度壓縮
  • 子對話 — 支援分支對話（Sub-conversation）
  • SSE 串流 — AI 回覆即時串流到前端
為什麼用 Go？因為即時串流需要處理大量並發連線，Go 的 goroutine 最適合！
🤖Bot API — 機器人大使館 Go▼
讓外部機器人能跟 Kuse 互動，API 格式相容 Telegram：
  • getMe — 取得機器人資訊
  • getUpdates — Long-polling 拿新訊息
  • sendMessage — 發送訊息
  • setWebhook — 設定 Webhook 回調
  • checkCredits — 檢查餘額
  • executeConnector — 執行連接器（整合外部服務）
還支援 WebSocket Hub，可以同時維持數千個即時連線！
💾FUSE 檔案系統 — 把雲端變成硬碟 Go 獨有▼
這是 Go 後端最酷的功能！讓 AI 機器人可以像操作本地硬碟一樣操作雲端檔案：
  • POSIX 相容 — 支援 chmod、chown、symlink
  • 即時讀寫 — 直接 read/write 檔案內容
  • 目錄操作 — mkdir、ls、rename
  • JWT 認證 — 每個 FUSE mount 用獨立 Token
Python 做不了這個！因為 FUSE 需要極低延遲的系統級操作。
🎯Skill System — 技能倉庫 Go▼
管理 AI 可以使用的各種「技能」：
  • V1 Skills — 基於檔案的技能定義
  • V2 Skills — 資料庫驅動，支援熱更新
  • 技能啟用/停用（per user）
  • 技能上傳（zip 格式）
  • MCP 整合 — Model Context Protocol 工具執行
☸️Claw / OpenClaw 管理 Go▼
管理 OpenClaw AI Agent Pod 的生命週期：
  • allocate — 分配新的 Pod
  • status — 檢查 Pod 狀態
  • redeploy — 重新部署
  • delete — 刪除 Pod
  • set-env — 更新環境變數
  • Admin API — 批量更新映像、批量重部署
📦Go Monorepo 裡的其他服務 ▼
  • gateway/ — API 閘道，統一入口
  • corg/ — 租戶/組織配置管理
  • wallet/ — Credits 錢包服務
  • library/ — 檔案儲存庫服務
  • insight/ — 分析洞察服務
  • llm_proxy/ — LLM API 代理
  • kuse-openclaw-orchestrator/ — OpenClaw 編排
  • kuse-fuse-client/ — FUSE 掛載客戶端

7
結果串流回前端 → 去背後的圖片顯示在畫布上 🎉前端
