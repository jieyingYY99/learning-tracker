The Hook

// hooks/useCompositionState.ts — dual check 双重检查
const composingRef = useRef(false); // ref, not state (no re-render)
const isComposing = (e) =>
  composingRef.current || e.nativeEvent.isComposing;
// ref guards browser race; native flag guards missed events

Deep Dive — Two Consumption Patterns 两种消费模式

A: Local instantiation 本地实例化 — each component calls hook directly. Simple, one input per component.

B: Prop delegation 父传子 — hook in parent, callbacks as props to memoized child. For virtual lists (AssetsTree) where only one row edits at a time.

React input (rename, search)? -> useCompositionState hook
Inside Tiptap? -> Native DOM listener + ref
Low-risk input? -> Inline nativeEvent.isComposing
Virtual list rows? -> Parent hook + props delegation

Pattern Exclusive View State 互斥视图状态

Pattern: 6 boolean flags made mutually exclusive via resetAllViews(). All enter*View() actions route through it.

6 个布尔标志通过 resetAllViews() 实现互斥。所有 enter*View() 操作都经过它。

// store/ui/index.ts
resetAllViews(merge?) {
  if (!anyFlagActive && !merge) return; // Early exit optimization
  set({
    isTrashView: false, isSharedWithMeView: false,
    isKuseClawView: false, isScheduledTasksView: false,
    isCustomizeView: false, isNewTaskView: false,
    ...merge  // Set new view atomically
  });
}

Exclusive state choice 互斥状态选型

2 states, no metadata? -> Boolean
3+ states, no metadata? -> String union enum (ViewMode)
3+ states, some with metadata? -> Booleans + resetAllViews
Max type safety? -> Discriminated union (higher migration cost)

Quiz 测验

Scenario-based: which approach would you choose?
场景题：你会选哪种方案？

Q1: Building a new "Favorites" feature — simple list, CRUD, no cross-component reactivity needed. Which store pattern?
新建"收藏"功能 — 简单列表、CRUD、无跨组件响应需求。选哪种 store？
Correct! MVP with no cross-component needs = simple store. Upgrade later if needed.
正确！无跨组件需求的 MVP 用简单 store，后续按需升级。
Q2: User typing Chinese in a Tiptap editor rename input presses Enter to confirm IME. How to prevent accidental submission?
用户在 Tiptap 编辑器中用中文输入法按 Enter 确认。如何防止意外提交？
Correct! ProseMirror intercepts keyboard events before React. Must use native DOM listeners.
正确！ProseMirror 在 React 之前拦截键盘事件，必须用原生 DOM 监听器。
Q3: Adding "Analytics" view to sidebar (alongside Trash, Shared, etc). No init data. How to wire it?
侧边栏新增 Analytics 视图。无初始化数据。如何接入？
Correct! Follow the pattern: route through resetAllViews for mutual exclusion. Direct set (A) allows two views active.
正确！遵循模式：通过 resetAllViews 路由。直接 set (A) 会导致两个视图同时激活。
Q4: Need a discoverable command picker in Tiptap that filters as user types. Which extension?
需要 Tiptap 中随输入过滤的命令选择器。用哪种扩展？
Correct! Suggestion provides live-filtering lifecycle. InputRule has no UI. ReactNodeView is for persistent widgets.
正确！Suggestion 提供实时过滤生命周期。InputRule 无 UI。ReactNodeView 用于持久组件。
Q5: Your Suggestion menu (ReactRenderer) needs Zustand store data. Will useStore() work?
你的 Suggestion 菜单（ReactRenderer）需要 Zustand store 数据。useStore() 能用吗？
Correct! Zustand stores are module singletons. useStore() hooks and getState() both work in isolated React roots. React Context providers (theme, auth) would NOT work.
正确！Zustand 是模块单例，useStore() 和 getState() 都能在独立 React root 中工作。React Context Provider 不行。
